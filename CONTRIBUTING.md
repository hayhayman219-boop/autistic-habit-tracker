# Contributing to Autistic Habit Tracker

Thanks for your interest in improving this project! This app is built around sensory-sensitive, autonomy-first design for autistic users, so contributions are especially welcome from autistic people, caregivers, and accessibility-focused developers.

## Ground rules

Please keep any changes consistent with the app's core design principles:
- **Calm, muted visuals** — no harsh contrast, no flashing or sudden motion
- **Respect for `prefers-reduced-motion`** in any new animation or transition
- **User autonomy** — habits and rewards should stay fully user-defined, never prescribed
- **Legible typography** — keep the Atkinson Hyperlegible font or an equally accessible alternative
- **Large, forgiving touch targets** for anyone with motor or coordination differences

## How to contribute

1. **Open an issue first** for anything beyond a small fix — describe the problem or idea so it can be discussed before you spend time on it.
2. **Fork the repository** and create a new branch for your change.
3. **Make your changes** in `autism-habit-tracker.html` (this is a self-contained single-file app — no build step required).
4. **Test in a browser** to confirm your change works and doesn't break existing functionality, especially data saved via `localStorage`.
5. **Submit a pull request** with a clear description of what changed and why.

## Reporting bugs or accessibility issues

Please open an issue and include:
- What you expected to happen
- What actually happened
- Your browser and device, if relevant
- Screenshots, if helpful

Accessibility issues (screen reader problems, color contrast, motion sensitivity, keyboard navigation, etc.) are a high priority — please flag them clearly as accessibility concerns.

## Code of conduct

Be respectful and patient. This project exists to help autistic people build routines on their own terms — keep that spirit in mind in all discussions and reviews.
